Social Dynamics — where the wolves live and how they group

Camera-survey pool only  ·  ? wolves analysed . Three views: spatial (which polygon), structural (which pack), and behavioural (pack / group / lone / unknown). Wolves marked with * are probable, not certain.

1. Per-polygon stacked by social dynamic

Each polygon's bar shows how its wolves split among pack / group / lone / unknown. Polygon name colour = land-use class. Hover for breakdown.

2. Pack inventory — sizes and home polygons

Each row is one identified pack. Solid bar = certain members, lighter hatched extension = probable members (asterisk). Bar colour = land-use class of the pack's home polygon (blue=Nature Reserve, orange=Minefield, pink=High Culling, green=Low Culling). Lone / no-pack wolves shown below the divider in neutral grey.

3. Land-use class — wolves per category, stacked by social

Polygons grouped by land-use class (per the paper's key). Nature Reserve = Yehodiya + Odem (most protected). Minefield = Makhfi + Saki + Hazeka (no humans). High Culling = Shaal + Snir + Keshet-Yonatan. Low Culling = Slopes + Firing-Zone + Masade.

4. Overall social-dynamic mix

All analysed (camera-survey) wolves, by canonical social category.

Pack Signatures — what unites and individualises within each pack

Each named pack (n ≥ 2): the pattern axis tends to individualise members; the colour and contrast axes tend to tag the pack. Each region is split by its grammar — A1/A2/C6/D8 → pattern + colour; B3/B4/B5 → pattern + a/b contrast; D9 → contrast level only; C7 → pattern only.

Top 3 packs per region — colour / contrast comparison

For each region: the three packs whose colour (A1/A2/C6/D8), contrast (B3/B4/B5/D9), or pattern (C7) is most consistent within the pack. Each vertical bar shows one pack's full within-pack distribution at the region's primary axis; the diagonal-striped grey segment marks wolves without info on this axis (N/P or code missing the relevant half), so each bar represents the whole pack. Packs with fewer than 3 observed wolves on that region are excluded so small-sample 100%s don't dominate.

Cross-pack colour signatures

Per pack × per colour-bearing region: modal colour (large), share, and 1–2 secondary colours. Cell background uses each region's reference palette (bubble shades from the colour-code maps).

Region-role map

Average within-pack modal % per region per axis. High → the axis consistently tags the pack; low → the axis individualises wolves.

Pack profile cards — per-region breakdown

Each card shows one pack. Bars are full within-pack distributions (sorted by frequency). Region rows tinted green = all members share an identical code.
About this page: Read-only by default. The Admin login (password protected) lets the data owner edit cells in the browser. Edits don't change the file on the server — clicking Save downloads an updated wolves_data.xlsx which the owner places back in the project folder. Running update.bat then refreshes all analyses and rebuilds this page.
Security note: the admin password is a soft client-side gate. It prevents accidental edits, not determined viewers. All data on this page is visible to anyone who can open it (this is by design — the table is meant to be shared).
